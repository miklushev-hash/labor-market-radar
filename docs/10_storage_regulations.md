# Регламент работы с Хранилищем

Этот документ фиксирует короткие, жесткие правила работы с файлом `Хранилище.xlsx`.

## 1. Один рабочий файл

Единственным рабочим файлом считается `Хранилище.xlsx`.

Внутри книги:

- лист `Недели` — для недель;
- лист `Месяцы` — для календарных месяцев;
- лист `Кварталы` — для кварталов;
- лист `Полугодия` — для полугодий;
- лист `Годы` — для календарных лет.
- лист `Журнал_событий` — для стратегических событий периода.

## 2. Технические копии не являются рабочими

Файлы с пометками `before`, `tmp`, `clean`, `fixed`, `broken` не считаются рабочим хранилищем.

## 3. Перед новым периодом делать копию

Перед новым периодом сохраняется копия:

- `Хранилище_before_fill_YYYY-WXX.xlsx`
- `Хранилище_before_fill_YYYY-MM.xlsx`
- `Хранилище_before_fill_YYYY-QX.xlsx`
- `Хранилище_before_fill_YYYY-H1.xlsx` или `Хранилище_before_fill_YYYY-H2.xlsx`
- `Хранилище_before_fill_YYYY.xlsx`

## 4. Не править книгу вручную вне Excel

Рабочая норма — редактировать книгу как книгу Excel.  
Ручная правка внутренней структуры файла допускается только как аварийное восстановление, а не как обычный процесс.

## 5. Один период = полный проход по всему реестру

Для каждого периода должна быть строка по каждому источнику из `Источники данных.xlsx`.

Пока по источнику нет строки и статуса, период не собран.

## 6. Источник считается обработанным только после решения

Нельзя считать источник проверенным только потому, что открылась главная страница.

Источник считается обработанным только если:

1. найден профильный раздел;
2. проверен нужный период;
3. выбран статус;
4. решение занесено в хранилище.

## 7. Отсутствие данных тоже фиксируется

Если релевантного материала нет, это тоже нужно записать.

Допустимые формулировки:

- `новых данных за период нет`
- `релевантных материалов за период не найдено`
- `требуется ручная проверка источника`

## 8. Сначала полный обход, потом текст

Порядок всегда один:

1. пройти весь реестр;
2. заполнить хранилище по каждому источнику;
3. сделать стоп-проверку полноты;
4. написать выпуск;
5. отметить использованные источники;
6. собрать HTML и PDF.

Писать выпуск до завершения полного обхода нельзя.

## 9. Поле `использован в обзоре` заполняется строго по факту

В поле `использован в обзоре` ставится `да` только если источник реально повлиял на текст выпуска.

Если материал найден, но в текст не вошел, ставится `нет`.

## 10. Приложение по источникам должно показывать весь реестр

В выпуске нельзя показывать только красивый список использованных источников.

Приложение по источникам должно отражать:

- весь реестр периода;
- статус каждого источника;
- краткий результат проверки;
- что именно вошло в выпуск.

## 11. Перед завершением периода нужна сверка

Перед завершением нужно проверить:

- у всех источников есть статус;
- у всех источников заполнены ключевые поля;
- у выпуска нет тезисов, которых нет в хранилище;
- всем реально использованным источникам проставлено `да`;
- приложение по источникам отражает весь реестр;
- HTML и PDF собраны по финальной версии текста.

## 12. При сбое не чинить поверх

Если файл ведет себя странно:

1. закрыть Excel;
2. сохранить проблемную копию отдельно;
3. не продолжать редактирование поверх поврежденной версии;
4. восстановить работу от последней нормальной копии.
