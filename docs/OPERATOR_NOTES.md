# Рабочие заметки по режиму агента

## Коротко

Этот файл нужен как короткая памятка о том, как агент должен работать внутри проекта.

## Что делает агент

Агент не просто пишет текст, а проходит полный цикл:

1. определяет период;
2. проходит весь реестр источников;
3. отдельно проверяет стратегические компании;
4. отдельно проверяет стратегические события;
5. заполняет `Хранилище.xlsx`;
6. пишет выпуск только по зафиксированному корпусу;
7. отмечает использованные источники;
8. собирает HTML и PDF.

## Какие режимы поддерживает проект

Сейчас проект поддерживает пять регулярных форматов:

- недельный срез;
- месячный обзор;
- квартальный обзор;
- полугодовой обзор;
- годовой обзор.

Кроме них, по запросу пользователя может собираться **сводный радар текущей ситуации**. Это не регулярный выпуск и не новый слой `Хранилище.xlsx`, а управленческая интерпретация уже готовых выпусков с числовыми индексами.

Радар стоит делать:

1. перед встречей с руководством;
2. когда накопилось несколько выпусков и нужна короткая управленческая картина;
3. когда появился сильный внешний или внутренний сигнал;
4. когда нужно принять решение по найму, удержанию, вилкам, развитию компетенций.

## Что агент не имеет права делать

- писать выпуск по удобному подмножеству источников;
- считать источник проверенным после открытия только главной страницы;
- добавлять тезис в выпуск, если его нет в хранилище;
- превращать слабый сигнал в сильный вывод;
- ставить числовые индексы в сводном радаре без объяснения и опоры на готовые выпуски;
- записывать сводный радар в `Хранилище.xlsx` или создавать для него новый лист;
- игнорировать стратегические события только потому, что они пришли не из классического HR-источника;
- удалять что-либо из папки `Корзина`: чистить `Корзина` имеет право только пользователь. Агент может только переносить туда лишние файлы по явному или очевидному рабочему правилу, но никогда не удаляет содержимое `Корзина` ни при каких условиях.
