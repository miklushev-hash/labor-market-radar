# Карта источников и недельный порядок работы

## Основная идея

В операционной работе мастер-списком считается файл `Источники данных.xlsx` в корне проекта.

Главное правило:

- проверяем все источники из этого файла;
- по каждому источнику принимаем решение;
- результат решения фиксируем в `Хранилище.xlsx`;
- только потом пишем выпуск.

## Что означает полная проверка источника

Полная проверка источника — это не открытие главной страницы.

Полная проверка означает:

1. найден профильный раздел источника;
2. проверен нужный период;
3. определено, есть ли сигнал;
4. решение занесено в хранилище.

Пока эти четыре пункта не выполнены, источник не считается обработанным.

## Что проверять внутри источника

В зависимости от типа источника нужно смотреть не только главную страницу, но и профильные разделы:

- новости;
- исследования;
- аналитика;
- блог;
- архив;
- пресс-центр;
- зарплаты;
- вакансии.

## Недельный порядок работы

1. Проверить все источники за прошлую полную неделю.
2. Сразу занести результат проверки каждого источника в `Хранилище.xlsx`.
3. Довести все источники до статуса.
4. Сделать стоп-проверку полноты обхода.
5. Очистить дубли по найденным релевантным материалам.
6. Разметить сигналы по темам, регионам, отраслям и контурам.
7. Проверить надежность сигналов.
8. Сравнить с предыдущими периодами.
9. Написать выпуск только по зафиксированному корпусу.
10. Вернуться в хранилище и отметить использованные источники.
11. Собрать HTML и PDF.

## Пять аналитических контуров

Каждый материал по возможности привязывается к одному из пяти контуров:

1. `world_event`
2. `global_macro`
3. `ru_macro`
4. `ru_labor`
5. `ru_it_labor`

## Приоритет источников

Если один и тот же тезис встречается в нескольких местах, доверяем в таком порядке:

1. официальная статистика и регуляторы;
2. крупные платформы с собственной методологией;
3. качественные деловые медиа;
4. Telegram.

## Правило фактчекинга

Каждый сигнал получает:

- `source_confidence`
- `verification_status`
- `evidence_type`
- `evidence_sources`

Сильные выводы можно делать только по сигналам со статусом `supported` или `confirmed`.

## Что должно получиться на выходе

- `Источники данных.xlsx` — реестр обязательных источников;
- `Хранилище.xlsx` — журнал результата проверки по каждому источнику;
- Markdown-выпуск;
- HTML;
- PDF;
- приложение по источникам, отражающее весь реестр периода.

## Ключевое организационное правило

Нельзя ограничиваться удобным набором источников, если реестр шире.  
Не каждый источник обязан попасть в текст, но каждый источник обязан быть доведен до решения.
