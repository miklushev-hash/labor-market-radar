# Как перенести проект на другой компьютер

## Цель

Этот файл нужен, чтобы перенести текущую рабочую версию проекта на другой компьютер без потери логики, структуры и дисциплины процесса.

## Что нужно перенести

Нужно копировать всю папку проекта целиком.

Особенно важны:

- `docs`
- `config`
- `taxonomy`
- `templates`
- `data`
- `presentations`
- `source`
- `Корзина`
- рабочие книги в корне проекта

## Какие режимы поддерживаются

Сейчас проект поддерживает:

- недельный срез;
- месячный обзор;
- квартальный обзор;
- полугодовой обзор;
- годовой обзор.

## Как выглядят выходные файлы

### Неделя

- `data/<week>/weekly_digest_multilayer.md`
- HTML
- PDF

### Месяц

- `data/<period>/monthly_digest.md`
- HTML
- PDF

### Квартал

- `data/<period>/quarterly_digest.md`
- HTML
- PDF

### Полугодие

- `data/<period>/halfyear_digest.md`
- HTML
- PDF

### Год

- `data/<period>/annual_digest.md`
- HTML
- PDF

## Какие правила нельзя потерять при переносе

1. Нельзя писать выпуск до полного обхода реестра.
2. Нужно проверять не только главную страницу, а все релевантные разделы и архивы.
3. Не каждый источник обязан войти в текст, но каждый обязан быть доведен до решения.
4. Стратегические события и стратегические компании проверяются отдельно.
5. Период считается завершенным только вместе с HTML, PDF и синхронизированным хранилищем.
6. `Корзина` переносится вместе с проектом, но очищать ее имеет право только пользователь.
