# Этап 2. Отбор только за прошлую неделю

## Цель
Автоматически выделять только релевантные материалы за нужное недельное окно.

## Правило окна дат
- Берём публикации только за прошлую полную неделю.
- Неделя хранится как:
  - `week_start`;
  - `week_end`;
  - `week_label` — например, `2026-W16`.

## Шаги обработки
1. Отфильтровать материалы по `published_at`.
2. Удалить дубли:
   - одинаковый URL;
   - одинаковый заголовок;
   - почти одинаковый текст;
   - репосты и копии из Telegram.
3. Разложить материалы по темам.

## Предлагаемые темы
- найм;
- увольнения и сокращения;
- зарплаты и компенсации;
- кадровый дефицит;
- рынок ИТ;
- миграция кадров;
- макроэкономика;
- корпоративные сигналы;
- регулирование и трудовое право;
- образование и навыки.

## Черновая схема классификации
```json
{
  "item_id": "uuid",
  "week_label": "2026-W16",
  "is_duplicate": false,
  "duplicate_group": null,
  "topic_primary": "salary",
  "topic_secondary": ["it_market", "hiring"],
  "language": "ru"
}
```

## Правила качества
- У материала должна быть определена дата публикации.
- Материал без текста или с очень коротким текстом идёт в отдельный список на проверку.
- Темы должны назначаться единообразно по словарю.
