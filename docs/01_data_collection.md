# Этап 1. Сбор данных

## Цель

Собрать единый входящий поток сигналов по рынку труда из открытых источников и отдельных стратегических контуров наблюдения.

## Категории источников

1. Сайты и медиа по рынку труда, найму, зарплатам, сокращениям, HR-аналитике.
2. Telegram-каналы по рынку труда, найму, зарплатам, сокращениям, кадровому дефициту.
3. Макроэкономические источники:
   - инфляция;
   - ключевая ставка;
   - безработица;
   - деловая активность;
   - данные по ИТ-сектору;
   - крупные корпоративные новости.
4. Компании стратегического наблюдения.
5. Стратегические события периода.

## Что фиксируем по каждому источнику

- `source_id` — уникальный идентификатор источника;
- `source_name` — название;
- `source_type` — тип источника;
- `country` — страна;
- `region` — регион, если применимо;
- `industry_focus` — отрасль или `multi`;
- `url` — основная ссылка;
- `access_method` — `rss`, `scrape`, `api`, `manual`, `telegram_export`;
- `update_frequency` — ожидаемая частота обновления;
- `priority` — важность источника;
- `notes` — ограничения, качество, особенности.

## Базовая логика

1. Вести реестр источников.
2. Отдельно вести реестр компаний стратегического наблюдения.
3. Отдельно вести справочник типов стратегических событий в `События стратегического наблюдения.xlsx`.
4. Журнал стратегических событий вести в листе `Журнал_событий` книги `Хранилище.xlsx`.
5. Для каждого источника определять способ получения данных.
6. Собирать не только текст, но и метаданные публикации.
7. Сразу сохранять дату публикации, чтобы потом фильтровать по периоду.
