# Этап 5. Выдача результата

## Цель

Передавать в итог не просто текст, а законченный выпуск в управленческом формате.

Эта логика одинаково действует для недели, месяца, квартала, полугодия и года. Меняются только период анализа, уровень доказательности и имя выходных файлов.

Управляющая матрица форматов находится в файле:

- [Каркас системы выпусков.xlsx](C:/Users/emiklush/Desktop/Codex/Обзор%20РТ/Каркас%20системы%20выпусков.xlsx)

При сборке комплекта выпуска матрица считается основным справочником по цели формата, структуре, обязательным сравнениям, итоговым файлам и месту хранения.

Актуальными визуальными версиями считаются HTML- и PDF-файлы, лежащие непосредственно в `presentations`. Файлы в `presentations/Архив` не используются для заполнения рабочего хранилища без отдельного запроса на работу с архивом.

## Что входит в итог

Итоговый результат всегда должен включать:

1. Markdown-файл выпуска;
2. HTML-версию;
3. PDF-версию;
4. приведенное в соответствие хранилище;
5. приложение по проверке источников, которое отражает весь реестр периода.

## Правило выбора формата

### Для недели

- если подтвержденный корпус узкий, итоговый формат — короткий недельный выпуск;
- если подтвержденный корпус широкий, итоговый формат — полный недельный обзор.

### Для месяца

- если подтвержденный корпус узкий, итоговый формат — короткий месячный выпуск;
- если подтвержденный корпус широкий, итоговый формат — полный месячный обзор.

### Для квартала

- если подтвержденный корпус узкий, итоговый формат — короткий квартальный выпуск;
- если подтвержденный корпус широкий, итоговый формат — полный квартальный обзор.

### Для полугодия

- если подтвержденный корпус узкий, итоговый формат — короткий полугодовой выпуск;
- если подтвержденный корпус широкий, итоговый формат — полный полугодовой обзор.

### Для года

- если подтвержденный корпус узкий, итоговый формат — короткий годовой выпуск;
- если подтвержденный корпус широкий, итоговый формат — полный годовой обзор.

Нельзя искусственно делать большой выпуск, если доказательная база тянет только на короткий формат.

## Принцип генерации текста

1. сначала факты;
2. потом механизм влияния;
3. потом интерпретация;
4. потом вывод для работодателя;
5. отдельно отметить, где есть подтвержденный тренд, а где только слабый сигнал.

## Стратегические событийные сигналы в итоговом выпуске

Если в периоде были события, которые меняют отраслевой, клиентский или корпоративный контекст, они не должны растворяться внутри обычного новостного потока.

Такие события должны подаваться как отдельные типы сигнала:

- стратегическое отраслевое событие;
- сигнал по клиенту;
- сигнал по крупному работодателю;
- сигнал замещения трудом ИИ.

Если в периоде есть хотя бы один такой подтвержденный сигнал, в выпуске должен появляться отдельный смысловой блок с его разбором.

## Компании стратегического наблюдения в итоговом выпуске

Если событие относится к компании стратегического наблюдения и влияет на наш рынок, на ИТ-контур или на значимого клиента, это должно быть явно отражено в выпуске.

Нельзя терять такие сигналы только потому, что они пришли не из стандартного HR-источника.

Основанием для такой проверки служит рабочий список:

При выпуске периода этот список должен быть просмотрен и отработан так же дисциплинированно, как и основной реестр источников.

## Обязательное приложение по источникам

В каждом выпуске должно быть приложение по источникам.

Оно обязано показывать не только использованные источники, но весь реестр периода:

- найден релевантный материал;
- новых данных нет;
- не релевантно;
- требуется ручная проверка;
- что именно вошло в текст.
