# Публичный прототип Labor Market Radar

## Что это

Это публичный прототип дашборда Labor Market Radar.

Он показывает:

- текущий режим рынка;
- индексы из `radar_snapshot.json`;
- главные сигналы;
- рекомендуемые действия;
- ограничения и ручные проверки;
- выпуски-основания, на которых собран snapshot.

Внутренний контур компании пока не подключен. Он отражен как ограничение продукта, а не как реальные HR-данные.

## Откуда берутся данные

Дашборд читает файл:

```text
../../data/current_radar/radar_snapshot.json
```

То есть данные больше не зашиты внутрь HTML как демонстрационный массив. `index.html` отвечает за интерфейс, а `radar_snapshot.json` - за текущую картину.

Если JSON не загрузился, дашборд показывает fallback-предупреждение и не должен считаться актуальным радаром.

## Как открыть

Публичная ссылка:

```text
https://miklushev-hash.github.io/labor-market-radar/prototypes/labor-market-radar/
```

Локально файл лучше открывать через простой HTTP-сервер, потому что браузеры могут блокировать загрузку JSON из `file://`.

## Как обновляются данные

Целевая схема:

```text
регулярные выпуски
  -> правила извлечения
  -> radar_snapshot.json
  -> дашборд
```

Сводный радар текущей ситуации в этой схеме опционален. Он может быть основанием для snapshot, если явно собран по запросу и содержит проверенные индексы, сигналы, действия и ограничения.

## Что проверять

1. Дашборд открывается по публичной ссылке.
2. На экране видны дата snapshot, статус и методология.
3. Значения индексов совпадают с `data/current_radar/radar_snapshot.json`.
4. Показаны сигналы, действия, ограничения и основания.
5. Если JSON недоступен, интерфейс явно показывает предупреждение.
